# 클로드 코드 — Hook: 강제로 실행되는 자동 가드레일

## 🎯 Hook이란 — 도구 호출 전후에 자동 실행

`Hook`(훅)은 **클로드가 도구(Tool)를 호출하기 직전·직후에 자동으로 끼어들어 실행되는 핸들러**입니다.

```
[PRE HOOK]            [TOOL CALL]              [POST HOOK]
위험 명령 차단  →   파일 수정·명령 실행   →   chime(알림)·lint
```

설정은 `.claude/settings.json` 파일에 적습니다. 심화 자동화에 강력하지만, 지금은 **"개념"**만 잡으면 충분합니다.

**비유:** Git의 **pre-commit 훅**처럼, 정해진 시점에 자동으로 끼어드는 **결정론적 핸들러**입니다. 사람이 깜빡해도 기계가 대신 챙겨줍니다.

---

## ⚖️ CLAUDE.md vs Hook — 부탁 vs 강제

이게 Hook의 핵심입니다: **지시는 무시될 수 있지만, 훅은 강제됩니다.**

| 구분 | [[20260613_클로드 코드 CLAUDE.md와 컨텍스트]] · **부탁** | Hook · **강제** |
|---|---|---|
| 성격 | "rm -rf 실행하지 마" 적어둬도 무시 가능 | 물리적으로 실행 차단 |
| 작동 | AI 지능에 의존 · 확률적 | 결정론적 · 실행 보장 |
| 위험 | 실수 시 복구 비용 큼 | 가드레일 + 자동 후처리 |

예를 들어 위험 명령(`rm -rf`)을 감지하면 **종료 코드 `exit 2`**를 내보내 해당 도구 실행을 **차단**합니다. CLAUDE.md에 "하지 마"라고 적는 것과 달리, 훅은 아예 실행을 막아버립니다.

## ⏱️ 이벤트는 3가지 '빈도'로 발생한다

| 빈도 | 이벤트 예시 |
|---|---|
| **세션당 1회** | `SessionStart` · `SessionEnd` |
| **턴당 1회** | `UserPromptSubmit` · `Stop` · `StopFailure` |
| **도구 호출마다** | `PreToolUse` · `PostToolUse` · `PermissionRequest` |

자주 쓰는 활용 6선:

| 이벤트 | 매처(어디에) | 활용 |
|---|---|---|
| `PreToolUse` | Bash·Edit·Write | 위험 명령 차단 · 파일 보호 |
| `PostToolUse` | Edit \| Write | 자동 lint · prettier · 테스트 |
| `UserPromptSubmit` | — | 입력 검증 · 위험 요청 사전 차단 |
| `Stop` / `StopFailure` | — | 완료 알림 · 자동 재시도 · 에러 처리 |
| `PreCompact` | manual \| auto | 컨텍스트 압축 전 백업 |
| `FileChanged` | .env \| .envrc | 민감 파일 감시 · 직접 편집 차단 |

## 🧱 settings.json 작성법 — 3단 구조

`이벤트 → 매처 → 훅` 3단으로 적습니다.

```json
// ~/.claude/settings.json
{
  "hooks": {
    "PreToolUse": [{          // 1) 이벤트: 언제
      "matcher": "Bash",      // 2) 매처: 어떤 도구에
      "hooks": [{             // 3) 훅: 무엇을
        "type": "command",
        "command": "... | grep 'rm -rf' && exit 2"
      }]
    }]
  }
}
```

**매처 규칙 3가지:**
- `*` · 빈 값 · 생략 → **모든 항목** 매칭
- `Bash` · `Edit|Write` → **정확한 문자열** 매칭
- 특수문자 포함 → **정규식** (예: `mcp__github__.*`)

`if` 조건으로 도구 인자까지 거를 수 있습니다 — `Bash(git *)`(git만), `Edit(*.ts)`(TypeScript 파일만), `Bash(rm *)`(rm 명령만).

> ⚡ **핫 리로드:** 설정 파일을 바꾸면 파일 감시자가 자동으로 다시 읽습니다 — 재시작 불필요.

## 🧪 직접 해보기 (개념 체크)

1. 내 작업 중 **"이건 절대 자동 실행되면 안 되는데"** 싶은 명령을 떠올린다 → `PreToolUse`로 막을 수 있음
2. 매번 손으로 하던 후처리(저장 후 정리·알림)를 떠올린다 → `PostToolUse`로 자동화 가능
3. "부탁(CLAUDE.md)으로 충분한가, 강제(Hook)가 필요한가"를 기준으로 구분해 본다

> ⚠️ **검토 요청:** 이벤트 이름·매처 문법·종료 코드 동작은 도구 버전마다 다를 수 있습니다. 실제 작성 전 최신 문서를 확인하세요.

## 💡 핵심 교훈

- Hook은 **도구 호출 전후에 강제 실행되는 결정론적 가드레일** — CLAUDE.md가 '부탁'이라면 Hook은 '강제'입니다.
- 이벤트는 **세션당 / 턴당 / 도구 호출마다** 3가지 빈도로 발생합니다.
- `settings.json`에 **이벤트 → 매처 → 훅** 3단 구조로 적고, `exit 2`로 위험 작업을 차단할 수 있습니다.
