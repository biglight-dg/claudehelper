# Git과 GitHub — 버전 관리와 협업의 기본

## 😱 "최종_진짜최종_v3_이게진짜.docx" 문제

문서를 고치다 보면 이런 파일이 쌓입니다. 어느 게 진짜 최신인지 아무도 모르죠. 이 혼란을 해결하는 도구가 **Git**입니다.

---

## 🕰️ Git — 시간을 되돌리는 '저장 기록기'

**Git**은 파일이 **언제, 무엇이, 어떻게 바뀌었는지 자동으로 기록**해주는 도구입니다. 게임의 **세이브 포인트**와 똑같아요.

- 작업하다 한 단계 끝나면 **저장(커밋, commit)**
- 잘못되면 **이전 저장 지점으로 되돌리기**
- 누가 무엇을 바꿨는지 **기록(히스토리)**이 다 남음

**비유:** 게임 세이브. 보스전에서 죽어도 마지막 세이브로 돌아가듯, Git이 있으면 언제든 안전한 시점으로 복귀할 수 있습니다. "최종_진짜최종"이 필요 없어집니다.

---

## ☁️ GitHub — Git을 위한 '공유 클라우드'

**GitHub**은 Git으로 관리하는 파일들을 **온라인에 올려두고 함께 작업**하는 공간입니다. Git이 '저장 기능'이라면, GitHub은 그걸 **올려두는 인터넷 창고 + 협업 공간**이에요.

- 내 작업을 올리기 = **푸시(push)**
- 남의 작업 받아오기 = **풀(pull)**
- 여럿이 같은 프로젝트를 **충돌 없이** 함께 수정

**비유:** Git이 '내 컴퓨터의 세이브 파일'이라면, GitHub은 '구글 드라이브 + 팀 작업실'. 전 세계 개발자가 코드를 공유하는 가장 큰 광장입니다.

| 구분 | Git | GitHub |
|---|---|---|
| 정체 | 도구(프로그램) | 웹 서비스 |
| 역할 | 변경 기록·되돌리기 | 온라인 공유·협업 |
| 위치 | 내 컴퓨터 | 인터넷(클라우드) |
| 비유 | 게임 세이브 | 공유 작업실 |

---

## 🧭 실무자에게 주는 의미

코드를 안 짜도 GitHub을 만날 일이 많습니다.
- AI 도구·오픈소스 프로그램이 대부분 GitHub에 공개돼 있어요.
- 문서·자료도 GitHub으로 버전 관리하는 팀이 늘고 있습니다.
- "이 프로젝트 GitHub 주소 줘" 같은 말을 알아듣고, 들어가서 **무슨 프로젝트인지 읽을 줄** 알면 충분합니다.

---

## 💡 핵심 교훈

- **Git** = 파일의 변경을 기록하고 되돌리는 **게임 세이브** 같은 도구.
- **GitHub** = Git 결과물을 올려 함께 작업하는 **온라인 공유 작업실**.
- 직접 쓰지 않아도, "버전 관리"와 "협업 창고"라는 두 개념만 잡으면 개발자와 소통이 됩니다.
