# 클로드 코드 — Skill: 반복 업무를 자동화하는 가장 경제적인 단위

## 🎯 Skill의 정체 — '절차서 + 스크립트' 묶음

`Skill`(스킬)은 **반복 업무 절차서(SOP)와 스크립트를 하나로 묶은 작업 매뉴얼**입니다. 매번 같은 방식으로 처리하므로, 반복이 많을수록 자동화 가치가 올라갑니다.

**⚠️ Skill ≠ Sub Agent (헷갈리지 마세요):**
- **Sub Agent** = 별도 context를 가진 '작업자' (일을 대신 해주는 사람)
- **Skill** = 부모(Parent)에게 주는 '작업 매뉴얼' (일하는 방법서)

구성은 `SKILL.md`(필수) + 선택 폴더 3개입니다.

---

## 🧰 Skill 구조 — 지휘자 + 도구함

| 요소 | 역할 | 비유 |
|---|---|---|
| `SKILL.md` (필수) | 절차서 — 무엇을, 어떤 순서로, 어떻게 | 🎼 지휘자 |
| `scripts/` (선택) | 실제 작업 실행 — API·파일 처리, 매번 동일 | 🎷 연주자 |
| `references/` (선택) | 가이드·톤·사례, 필요할 때 불러 읽는 지식 | 📚 참조 자료 |
| `assets/` (선택) | 템플릿·이미지·폰트·로고, 산출물에 그대로 | 🖼 정적 파일 |

**결합의 힘:** `scripts + assets`로 **절차적 안정성**을, `SKILL.md + references`로 **AI 유연성**을 동시에 얻습니다.

---

## ⚡ Skill vs MCP — 토큰 효율 차이

스킬이 '가장 경제적인 자동화 단위'로 불리는 이유는 **토큰 효율** 때문입니다.

| 방식 | 대화 시작 시 점유 토큰 | 특징 |
|---|---|---|
| **MCP** | 도구 전체를 미리 로드 (큰 통째 로드) | 대화 시작 전부터 컨텍스트를 크게 차지 |
| **Skill** | frontmatter(요약)만 먼저 로드 | 본문은 **호출 시점에만** 로드 ✓ |

스킬은 맨 앞 요약(frontmatter)만 항상 로드하고, 실제 절차 본문은 그 스킬을 쓸 때만 읽어옵니다. 그래서 컨텍스트를 거의 차지하지 않습니다. ([[20260612_토큰과 컨텍스트 아끼기]]의 원리와 같습니다.)

> ⚠️ **검토 요청:** 강의자료는 "MCP 방식 약 20,000 토큰 vs Skill frontmatter <100 토큰(약 200배)"으로 예시를 듭니다. 구체 수치는 도구·구성마다 크게 다르니 '경향'으로만 받아들이세요. 실무 팁: **MCP로 먼저 테스트 → 작동 확인 → Skill로 전환**.

## 💰 n8n을 대체한다 — 운영 사례

스킬은 SOP + 체크리스트 + 코드를 결합하고, 클로드의 지능으로 오류까지 처리해 **노코드 자동화 도구(n8n 등)를 대체**할 수 있습니다.

| 사례 | 하는 일 |
|---|---|
| 뉴스레터 작성 | 매주 같은 톤으로 작성 → 발송 |
| 계약서 자동화 | 문서에서 변수 추출 → JSON |
| YouTube → 블로그 | 영상 스크립트를 글로 변환 (톤 일관) |
| 알림톡 발송 | 발송 API 호출 자동화 |

([[20260612_노코드 자동화 워크플로]]와 비교하면, 스킬은 'AI가 직접 들고 일하는 매뉴얼'에 가깝습니다.)

## 🛠️ Skill 제작 7단계 — 완벽 대신 반복 개선

처음부터 완벽하게가 아니라, **반복하며 정확도를 70% → 90% → 98%로 끌어올립니다.**

1. 반복 업무 1개 선택
2. 기존 SOP·체크리스트 작성
3. 클로드에 'Skill 형식으로 변환' 요청
4. `SKILL.md` 작성
5. 필요한 `scripts` 작성
6. **Fresh session(새 세션)**에서 테스트
7. 실패 사례 → `SKILL.md`에 반영 (× N회)

**클로드에게 스킬을 요청하는 프롬프트 (검증 루프 포함):**
```
I want to build a skill that does X.
Research the skill format if needed,
then create the SKILL.md and any scripts required.
Test it in a fresh Claude Code instance
and update the skill based on failures.
```
핵심 키워드 4개: **목표(does X) · 형식 자가 학습(research format) · 검증 루프(test in fresh) · 반복 개선(update on failures).**

| # | 스킬 | 하는 일 |

## 🧪 직접 해보기

1. 내가 **매주/매일 똑같이 반복하는 업무** 1개를 고른다
2. 그 절차를 번호 매겨 체크리스트로 적는다 (= SOP)
3. 위 프롬프트로 클로드에게 `SKILL.md` 초안을 만들어 달라고 한다
4. 새 세션에서 돌려보고, 막힌 부분을 `SKILL.md`에 한 줄씩 추가한다

---

## 💡 핵심 교훈

- Skill은 **반복 업무 절차서 + 스크립트 묶음** — Sub Agent(작업자)가 아니라 매뉴얼입니다.
- `SKILL.md`(필수) + `scripts·references·assets`(선택)로 **안정성과 유연성**을 함께 얻습니다.
- **frontmatter만 먼저 로드**돼 토큰 효율이 높고, 반복 테스트로 정확도를 키우는 게 정석입니다.
