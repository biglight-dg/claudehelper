# 자동화용 Agent 설계 — 나만의 AI 일꾼 만들기

## 🧑‍💼 Agent를 '신입사원'처럼 설계하기

Agent(에이전트) 개념은 이미 배웠습니다(Agent와 Hook 참고). 이번엔 **실제로 내 업무 Agent를 설계**하는 법입니다. 핵심 비유는 하나 — **신입사원에게 업무 인수인계하듯** 설계하면 됩니다.

신입에게 일을 맡길 때 우리가 알려주는 것:
1. 너는 어떤 역할이야 (직무)
2. 무슨 일을 해야 해 (목표)
3. 어떤 순서·규칙으로 (절차)
4. 결과는 어떤 모양으로 (산출물)
5. 막히면 어떻게 (예외 처리)

Agent 설계도 똑같이 이 5가지를 적어주는 일입니다.

---

## 📋 Agent 설계 5요소

| 요소 | 질문 | 예시 |
|---|---|---|
| **역할** | 누구처럼 일할까 | "주간보고 정리 담당자" |
| **목표** | 무엇을 이뤄야 하나 | "팀원 진행상황 → 보고서" |
| **절차** | 어떤 순서로 | "수집 → 요약 → 양식 채우기" |
| **도구** | 무엇을 쓸 수 있나 | "팀 문서 폴더, 보고서 템플릿" |
| **규칙·예외** | 지킬 선, 막힐 때 | "수치 없으면 빈칸+표시" |

이 5요소를 한 장에 적으면 그것이 바로 **Agent 명세서**입니다. (이 교육 시스템의 교육자·QA·큐레이터도 이렇게 역할이 정의돼 있습니다.)

---

## ✍️ 실전 예시 — '회의록 정리 Agent'

```
역할: 회의록 정리 전문가
목표: 거친 회의 메모를 표준 회의록으로 변환
절차:
  1. 메모에서 논의/결정/할일 분류
  2. 할일은 담당자·기한 추출
  3. 회의록 템플릿에 채우기
도구: 회의록 마크다운 템플릿
규칙: 담당자·기한 불명확하면 '(확인 필요)' 표시
산출물: 마크다운 회의록 1장
```

이렇게 한 번 정의해두면, 매번 "이렇게 정리해줘"를 반복할 필요 없이 **"회의록 Agent 돌려줘"** 한마디로 끝납니다.

---

## 🪜 작게 시작해 키우기

처음부터 완벽한 Agent를 만들려 하지 마세요.
1. **단순 작업 하나**로 시작 (예: 요약만)
2. 돌려보고 **부족한 규칙을 추가**
3. 잘 되면 **절차를 한 단계씩 확장**

**비유:** 신입사원도 처음엔 작은 일부터 맡기며 키웁니다. Agent도 똑같이 **키워가는** 것입니다.

---

## 💡 핵심 교훈

- Agent 설계 = **신입사원 인수인계.** 역할·목표·절차·도구·규칙 5요소를 적어주면 됩니다.
- 한 번 정의하면 **반복 지시 없이 한마디로 재사용** — 그게 자동화의 핵심 이득.
- **작게 시작해 규칙을 더해가며 키우기.** 처음부터 완벽할 필요 없습니다.
