# 터미널이 뭘까 — 검은 화면에 대한 두려움 없애기

## ⬛ 터미널이라는 검은 화면

영화 속 해커가 두드리는 그 **검은 화면**, 글자만 가득한 그곳이 바로 **터미널(Terminal)**입니다. 다른 이름으로 **명령 프롬프트, 콘솔, CLI** 라고도 불러요.

무서워 보이지만, 정체는 의외로 단순합니다. **컴퓨터에게 글자로 명령을 내리는 창**일 뿐이에요.

---

## 🖱️ 마우스(GUI) vs ⌨️ 터미널(CLI)

우리가 평소 쓰는 방식은 **GUI** — 아이콘을 마우스로 누르는 방식입니다. 터미널은 **CLI** — 글자로 직접 명령하는 방식이에요.

**비유:** 식당에서 주문하는 두 가지 방법.
- **GUI(마우스):** 사진 메뉴판을 손가락으로 가리키기. 쉽지만 사진에 있는 것만 시킬 수 있어요.
- **CLI(터미널):** 직원에게 말로 "스테이크 미디움, 소금 빼고, 소스는 따로" 주문하기. 익숙해지면 **훨씬 정확하고 빠르고 세밀**합니다.

| 구분 | GUI (마우스) | CLI (터미널) |
|---|---|---|
| 방법 | 아이콘 클릭 | 글자 명령 입력 |
| 장점 | 쉽고 직관적 | 빠르고 정밀, 자동화 가능 |
| 단점 | 반복 작업이 번거로움 | 처음엔 낯섦 |
| 비유 | 사진 메뉴 가리키기 | 말로 세밀하게 주문 |

---

## ⌨️ 명령은 이렇게 생겼어요

터미널 명령은 대부분 **"동사 + 대상"** 구조입니다. 사람 말과 똑같아요.

| 명령 | 뜻 | 사람 말로 |
|---|---|---|
| `cd 폴더이름` | change directory | "그 폴더로 들어가" |
| `ls` (맥) / `dir` (윈도우) | list | "여기 뭐 있는지 보여줘" |
| `mkdir 이름` | make directory | "새 폴더 만들어" |

명령 한 줄을 치고 **Enter**를 누르면 컴퓨터가 실행합니다. 틀려도 괜찮아요 — 대부분의 명령은 "그런 거 없는데요" 하고 알려줄 뿐, 컴퓨터가 망가지지 않습니다.

---

## 🤝 그런데 왜 실무자가 터미널을 알아야 할까

좋은 소식: **이제는 터미널 명령을 외울 필요가 거의 없습니다.** Claude Code 같은 AI 도구가 **터미널 안에서** 동작하기 때문이에요.

즉, 우리는 검은 화면에 **한국어로 "이 폴더 정리해줘"라고 말만 하면**, AI가 알아서 적절한 명령을 실행해 줍니다. 터미널은 이제 "AI와 내 컴퓨터가 만나는 작업실"인 셈입니다.

**비유:** 예전엔 외국에서 직접 현지어로 주문해야 했지만, 지금은 통역사(AI)를 옆에 두고 한국어로 말하면 됩니다. 그래도 "여기가 식당이고 저 사람이 통역사구나" 하는 **기본 구조는 알아야** 편하게 시킬 수 있어요.

---

## 💡 핵심 교훈

- 터미널 = **글자로 컴퓨터에 명령하는 창**. 해킹 도구가 아니라 그냥 또 다른 주문 방식입니다.
- 명령은 "동사 + 대상" 구조라 사람 말과 닮았고, 틀려도 컴퓨터는 안 망가집니다.
- AI 시대엔 명령을 외울 필요 없이 **한국어로 말하면** AI가 대신 실행 — 그래서 더더욱 두려워할 이유가 없습니다.
