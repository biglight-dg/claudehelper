# 클로드 코드 — CLAUDE.md와 컨텍스트: 매번 자동 주입되는 '첫 지침서'

## 🎯 CLAUDE.md의 정체 — 프로젝트별 '사규'

`CLAUDE.md`는 **클로드 코드가 세션을 시작할 때마다 첫 메시지보다 먼저 자동으로 읽는 마크다운 파일**입니다. 매번 복사·붙여넣기 할 필요 없이, 그 프로젝트의 규칙을 자동으로 적용해 줍니다.

- **일반 대화 구조:** `User → Model → User → Model …`
- **클로드 코드 구조:** `CLAUDE.md → User → Model → User → Model …`

즉 CLAUDE.md는 **프로젝트마다 따로 거는 system prompt(운영 지침)**입니다. 한 번 적어두면 모든 세션에 자동 적용되고, Git으로 이력 추적까지 됩니다.

**비유:** 신입사원 책상에 붙여두는 **'업무 매뉴얼 한 장'**. 매번 말로 설명하지 않아도, 일을 시작하면 알아서 그 규칙대로 움직입니다. 앞서 배운 [[20260613_AI 지침과 md 파일 활용법]]의 '사규' 개념을 클로드 코드에서 실제로 구현한 것이 바로 이 파일입니다.

---

## 🧭 왜 중요한가 — 배의 항로 비유

출발할 때의 **작은 각도 차이가 도착지를 완전히 바꿉니다.**

- **단거리 항해(짧은 대화):** 1~2도 틀어져도 도착지가 크게 안 바뀜 → CLAUDE.md 영향 작음
- **대서양 횡단(긴 자동화):** 1도 차이가 수백 km 오차 → **초반 지침이 최종 결과를 결정**

CLAUDE.md가 없으면 경로가 사방으로 **발산**하고, CLAUDE.md를 적용하면 경로가 목표를 향해 **좁아집니다.** 작업이 길고 자동화될수록 첫 지침의 중요성이 커집니다.

---

## 📜 지식창고가 아니라 '운영 계약서'다

가장 흔한 실수는 CLAUDE.md를 **위키처럼** 쓰는 것입니다. 모든 지식을 욱여넣어 10,000단어짜리로 만들면, 오래된 규칙이 쌓여 오히려 방해가 됩니다.

| 구분 | ✗ 자주 하는 실수 | ✓ 올바른 사용 |
|---|---|---|
| **분량** | 10,000단어 위키 | 200~500줄 이내 |
| **성격** | 모든 지식 저장소 | 운영 계약서(움직이는 규칙) |
| **내용** | 오래된 규칙 누적 | 매번 같은 기준이 되는 핵심만 |

핵심은 **"무엇을 아는가"가 아니라 "어떻게 움직여야 하는가"**를 적는 것입니다.

---

## 🧱 좋은 CLAUDE.md의 5개 섹션

가장 중요한 규칙을 **맨 위**에 둡니다.

| 순서 | 섹션 | 무엇을 적나 |
|---|---|---|
| 01 | **Project Context** | 무엇을 만드는 프로젝트인가 |
| 02 | **Build & Test** | 빌드·실행·테스트 명령어 |
| 03 | **Boundaries** | 절대 건드리지 말 것 (DB·결제·인증) |
| 04 | **Safety** | 작업 전 사용자 확인이 필요한 항목 |
| 05 | **Style** | 말투·코드 스타일·출력 형식 |

**왜 맨 위가 중요한가 (편집 편향):**
- **Primacy bias** — 처음 본 정보를 잘 기억함
- **Recency bias** — 마지막에 본 정보를 잘 기억함

→ 그래서 **가장 중요한 규칙은 맨 위(또는 맨 아래)**에 배치합니다. 중간에 파묻으면 잊히기 쉽습니다.

---

## 📍 두 가지 위치만 기억하기

| 위치 | 적용 범위 |
|---|---|
| `프로젝트/CLAUDE.md` | **그 프로젝트에만** 적용 |
| `~/.claude/CLAUDE.md` | **내 모든 프로젝트에** 적용 |

이름은 정확히 **`CLAUDE.md`** — 대문자 그대로, 공백 없이. 세팅은 30초면 됩니다. 텍스트 에디터로 빈 파일 하나 만들고, 일러두고 싶은 걸 평문으로 적으면 끝입니다.

**⚠️ 없이 쓰면?** 매 세션이 'Zero Memory'에서 시작합니다 — 내가 누군지, 무슨 일을 하는지, 원하는 답 형식이 뭔지 매번 처음부터 다시 설명해야 합니다.

## 🧪 직접 해보기

1. 내 프로젝트 폴더에 빈 `CLAUDE.md` 파일을 만든다
2. 위 5개 섹션 중 **Project Context / Style** 두 줄만 먼저 적는다 (작게 시작)
3. 클로드 코드를 켜서, 시키지 않아도 그 규칙을 따르는지 확인한다
4. 매번 반복 설명하던 게 있으면 한 줄씩 추가 — **200~500줄을 넘지 않게**

> ⚠️ **검토 요청:** 파일을 자동으로 읽는 정확한 이름·위치·우선순위는 도구와 버전마다 다를 수 있습니다(Claude Code 기준). 실제 적용 시 최신 안내를 확인하세요.

## 💡 핵심 교훈

- CLAUDE.md는 **매번 자동 주입되는 프로젝트별 system prompt** — 복사 없이 모든 세션에 규칙이 적용됩니다.
- **지식창고(위키)가 아니라 운영 계약서** — 200~500줄, "어떻게 움직일지"만 적습니다.
- **5개 섹션 + 중요한 규칙은 맨 위**(Primacy/Recency bias). 긴 자동화일수록 첫 지침이 결과를 좌우합니다.
