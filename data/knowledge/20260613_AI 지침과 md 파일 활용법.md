# AI 지침과 .md 파일 활용법 — AI에게 '사규'를 만들어주기

## 🎯 .md 파일의 진짜 힘 — AI에게 주는 지침서

지금까지 `.md` 파일을 '깔끔한 문서'로 배웠습니다. 하지만 진짜 위력은 따로 있어요. **AI 도구에게 "이렇게 일해줘"라고 건네는 지침서(사규)**가 된다는 점입니다. 한 번 적어두면 매번 설명할 필요가 없어집니다.

**비유:** 신입사원에게 매번 말로 설명하는 대신, **'업무 매뉴얼' 한 장**을 책상에 붙여두는 것. AI 지침 파일이 바로 그 매뉴얼입니다.

---

## 🔁 다시, 마크다운 vs .md 파일 — 최종 정리

| 구분 | 정체 | 한마디로 |
|---|---|---|
| **마크다운** | 글쓰기 방식(문법) | "쓰는 법" |
| **.md 파일** | 그 방식으로 쓴 파일 | "저장된 결과물" |
| **AI 지침 .md** | AI가 읽고 따르는 .md 파일 | "AI에게 주는 사규" |

즉 같은 `.md` 파일이라도, **AI 도구에게 건네는 순간 '지침서'로 격이 올라갑니다.**

---

## 📄 실제 지침 파일은 이렇게 생겼습니다

아래는 'AI에게 주는 사규' 한 장의 예시입니다. (Claude Code는 보통 `CLAUDE.md`라는 이름의 파일을 자동으로 읽습니다.)

```
# 우리 팀 보고서 작성 지침

## 역할
너는 우리 팀의 보고서 정리 담당자야.

## 말투·형식
- 존댓말, 간결한 문장
- 제목은 ##, 핵심은 **굵게**
- 모든 보고서는 '요약 → 상세 → 다음 단계' 순서

## 금지
- 추측으로 수치 채우지 말 것 → 없으면 '(확인 필요)'
- 고객 실명 노출 금지 → '고객A'로

## 산출물
- 마크다운(.md) 한 장으로
```

이 파일을 프로젝트 폴더에 두면, AI가 작업할 때마다 **매번 이 규칙을 자동으로 따릅니다.** "존댓말로 써줘"를 다시 말할 필요가 없어요.

---

## 🧱 좋은 지침 파일의 4요소

| 요소 | 무엇을 적나 | 예시 |
|---|---|---|
| **역할** | AI를 누구로 둘지 | "보고서 정리 담당자" |
| **형식·말투** | 출력 모양과 톤 | "존댓말, 마크다운, 3단 구성" |
| **금지·규칙** | 하지 말아야 할 것 | "추측 수치 금지" |
| **산출물** | 최종 결과 형태 | ".md 한 장" |

'자동화용 Agent 설계'의 5요소와 닮았죠? **지침 파일은 그 설계를 .md로 적어둔 것**입니다.

---

## 🧪 직접 해보기

1. 내가 AI에게 **매번 반복하던 설명**을 떠올린다 ("존댓말로", "이 양식으로"…)
2. 위 4요소로 **지침 .md 한 장**을 적는다
3. 그 내용을 AI에게 건네고(또는 지침 파일로 두고) 결과가 달라지는지 확인
4. 부족하면 규칙을 한 줄씩 추가 — **작게 시작해 키우기**

> ⚠️ **검토 요청:** 지침 파일의 정확한 파일명·인식 방식은 도구(Claude Code 등)와 버전마다 다릅니다. 실제 적용 시 해당 도구의 최신 안내를 확인하세요.

## 💡 핵심 교훈

- `.md` 파일은 단순 문서를 넘어 **AI에게 주는 '사규·매뉴얼'**이 됩니다 — 한 번 적으면 매번 설명 불필요.
- 좋은 지침은 **역할·형식·금지·산출물** 4요소. (Agent 설계 5요소와 같은 뿌리)
- **마크다운=쓰는 법, .md=파일, AI 지침 .md=AI에게 주는 사규** — 이 셋의 구분이 이 과정의 결론입니다.
