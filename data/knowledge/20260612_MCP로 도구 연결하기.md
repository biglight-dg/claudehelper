# MCP로 도구 연결하기 — AI에게 손발을 달아주는 표준

## 🔌 MCP가 뭘까 — AI 세계의 '멀티탭'

**MCP(Model Context Protocol)**는 **AI와 외부 도구를 연결하는 표준 약속**입니다. AI가 채팅만 하는 게 아니라, **달력·파일·디자인 도구·검색** 같은 실제 도구를 쓸 수 있게 이어주는 통로예요.

**비유:** 멀티탭(콘센트 확장). AI라는 본체에 멀티탭(MCP)을 꽂으면, 거기에 달력 플러그·파일 플러그·디자인 플러그를 줄줄이 연결할 수 있습니다. 표준 규격이라 **어떤 도구든 같은 방식으로** 꽂혀요.

---

## 🤔 왜 MCP가 필요할까

예전엔 AI가 새 도구를 쓰려면 **도구마다 따로따로** 연결 작업을 해야 했습니다. 도구가 100개면 100가지 방식이 필요했죠. MCP는 이걸 **하나의 표준**으로 통일했습니다.

**비유:** 나라마다 다른 콘센트 모양 때문에 여행할 때 어댑터를 잔뜩 챙겨야 했는데, **전 세계 공통 콘센트**가 생긴 셈입니다. 한 번 규격을 맞추면 어떤 도구든 쉽게 연결됩니다.

---

## 🛠️ MCP로 할 수 있는 일 (예시)

이 교육 시스템도 MCP로 **Canva(디자인 도구)**와 연결돼 있어, AI가 직접 슬라이드를 만들 수 있습니다.

| 연결 대상 | AI가 할 수 있게 되는 일 |
|---|---|
| 파일 시스템 | 내 문서 읽고 정리 |
| 디자인 도구(Canva 등) | 슬라이드·이미지 제작 |
| 검색·웹 | 최신 정보 가져오기 |
| 캘린더·메일 | 일정 확인, 초안 작성 |
| 데이터베이스 | 자료 조회·정리 |

---

## 🧭 실무자가 기억할 것

- MCP를 **직접 설치·설정할 필요는 없습니다.** 보통 도구가 미리 연결해둡니다.
- 다만 "AI가 이 도구랑 **MCP로 연결돼 있다**"는 말을 들으면, **"아, AI가 저 도구를 직접 쓸 수 있구나"**로 이해하면 됩니다.
- 자동화를 설계할 때 **"이 도구를 AI에 연결할 수 있나?"**를 떠올리는 것 — 그게 핵심입니다.

---

## 💡 핵심 교훈

- **MCP** = AI와 외부 도구를 잇는 **표준 멀티탭**. AI에게 손발(도구 사용 능력)을 달아줍니다.
- 도구마다 제각각이던 연결을 **하나의 공통 규격**으로 통일한 것이 핵심 가치.
- 직접 설정하진 않아도, **"MCP로 연결됨 = AI가 그 도구를 직접 쓸 수 있음"**만 알면 충분합니다.
