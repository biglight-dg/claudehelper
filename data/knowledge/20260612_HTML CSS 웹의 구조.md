# HTML · CSS · JavaScript — 웹페이지를 이루는 3요소

## 🏠 웹페이지는 '집' 한 채

우리가 보는 모든 웹페이지는 **세 가지 재료**로 지어집니다. 집짓기에 비유하면 딱 맞아요.

| 재료 | 역할 | 집 비유 |
|---|---|---|
| **HTML** | 뼈대·내용 | 벽·기둥·방 배치 (골조) |
| **CSS** | 꾸미기·디자인 | 페인트·벽지·가구 (인테리어) |
| **JavaScript** | 움직임·기능 | 전등 스위치·자동문 (전기설비) |

이 셋이 합쳐져 우리가 보는 화면이 됩니다.

---

## 🦴 HTML — 내용과 뼈대

**HTML**은 "여기는 제목, 여기는 문단, 여기는 이미지"처럼 **무엇이 있는지**를 적는 뼈대입니다. 마크다운과 사촌지간이에요 — 둘 다 **태그(꼬리표)**로 구조를 표시합니다.

```html
<h1>큰 제목</h1>
<p>이것은 한 문단입니다.</p>
```

`<h1>`은 제목, `<p>`는 문단이라는 뜻. 여는 태그 `<p>` 와 닫는 태그 `</p>` 로 감쌉니다.

**비유:** HTML만 있는 페이지는 페인트칠 안 한 골조 건물 — 내용은 다 있지만 밋밋합니다.

---

## 🎨 CSS — 디자인과 꾸미기

**CSS**는 그 뼈대를 **예쁘게** 만듭니다. 색, 글꼴, 크기, 간격, 배치를 담당해요.

```css
h1 { color: blue; font-size: 32px; }
```

"제목(h1)은 파란색, 32픽셀 크기로". 같은 HTML이라도 CSS에 따라 완전히 다른 분위기가 됩니다.

**비유:** 같은 골조 건물도 인테리어(CSS)에 따라 카페가 되기도, 사무실이 되기도 합니다.

---

## ⚡ JavaScript — 움직임과 기능

**JavaScript(JS)**는 페이지에 **동작**을 넣습니다. 버튼을 누르면 메뉴가 펼쳐지고, 좋아요를 누르면 숫자가 올라가는 것 — 전부 JS입니다.

**비유:** 전기 설비. 스위치를 누르면 불이 켜지듯, 사용자가 무언가 하면 페이지가 반응하게 만듭니다.

---

## 🧭 실무자가 알아야 할 딱 이만큼

직접 코딩할 필요는 없습니다. 다만 이 구분을 알면:
- 웹 작업을 외주 줄 때 "이건 HTML 문제(내용)인지, CSS 문제(디자인)인지" 소통이 됩니다.
- AI에게 "이 페이지 색만 바꿔줘(CSS)" 처럼 **정확히 부탁**할 수 있습니다.

---

## 💡 핵심 교훈

- 웹페이지 = **HTML(뼈대) + CSS(디자인) + JavaScript(동작)** 세 재료로 지은 집.
- HTML은 마크다운의 사촌 — **태그로 구조**를 표시합니다.
- 직접 만들진 않아도, **"내용·디자인·동작" 세 영역을 구분**할 줄 알면 소통과 요청이 정확해집니다.
