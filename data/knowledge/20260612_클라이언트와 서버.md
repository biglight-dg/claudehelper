# 클라이언트와 서버 — 인터넷은 어떻게 동작하나

## 🍽️ 식당으로 이해하는 인터넷

인터넷에서 일어나는 거의 모든 일은 **'손님이 주문하고 주방이 만들어 내보내는'** 구조입니다.

- **클라이언트(Client)** = 손님. 내 폰·노트북의 앱·브라우저.
- **서버(Server)** = 주방. 멀리 있는 큰 컴퓨터.
- **요청(Request)** = 주문. "이 페이지 주세요"
- **응답(Response)** = 음식 서빙. 서버가 보내주는 결과.

유튜브를 켜는 순간: 내 폰(클라이언트)이 "영상 줘"라고 **요청**하면, 유튜브 컴퓨터(서버)가 영상을 **응답**으로 보내줍니다.

---

## 🔁 요청–응답의 한살이

```
[클라이언트]  ──── 요청(Request) ────▶  [서버]
   내 브라우저                            먼 곳의 컴퓨터
[클라이언트]  ◀──── 응답(Response) ────  [서버]
```

이 왕복이 1초에도 수십 번 일어납니다. 웹페이지 하나 열 때 글자·이미지·동영상을 각각 따로 요청하거든요.

---

## 🖥️ 서버는 특별한 컴퓨터일까

서버도 그냥 **컴퓨터**입니다. 다만 **항상 켜져 있고, 여러 손님을 동시에 받도록** 만들어진 컴퓨터예요.

**비유:** 우리 집 부엌(개인 PC)도 요리는 되지만, 식당 주방(서버)은 24시간 돌아가며 수백 명을 동시에 먹입니다. 같은 '부엌'이지만 규모와 역할이 다른 거죠.

| 구분 | 클라이언트 | 서버 |
|---|---|---|
| 역할 | 요청하는 쪽 | 응답하는 쪽 |
| 비유 | 손님 | 주방 |
| 예 | 내 브라우저·앱 | 유튜브·네이버 컴퓨터 |
| 상태 | 필요할 때 켬 | 늘 켜져 있음 |

---

## ☁️ "클라우드"도 결국 서버

요즘 흔한 **클라우드(cloud)**는 "남의 서버를 빌려 쓰는 것"입니다. 내가 주방을 직접 차리는 대신, **잘 갖춰진 공유 주방을 빌리는** 셈이에요. 구글 드라이브에 파일을 올리면, 그 파일은 내 컴퓨터가 아니라 구글의 서버(클라우드)에 저장됩니다.

---

## 💡 핵심 교훈

- 인터넷 = **클라이언트(손님)**가 **요청**하면 **서버(주방)**가 **응답**하는 식당 구조.
- 서버도 그냥 컴퓨터 — 단지 **늘 켜져 있고 여러 손님을 받는** 컴퓨터.
- **클라우드 = 남의 서버 빌려 쓰기.** 내 파일이 '어딘가의 서버'에 있다는 감각만 잡으면 충분합니다.
