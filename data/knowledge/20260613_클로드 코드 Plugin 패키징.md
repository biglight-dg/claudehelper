# 클로드 코드 — Plugin: 스킬·훅·MCP를 한 패키지로 묶어 재사용

## 🎯 플러그인이란 — 묶어서 한 번에 설치

`Plugin`(플러그인)은 **스킬·서브에이전트·훅·MCP·기본 설정을 하나로 묶을 수 있는 확장 단위**입니다. `.claude-plugin/plugin.json` 매니페스트를 중심으로 구성됩니다.

**왜 필요한가?** 스킬·훅·MCP를 프로젝트마다 따로 설정하다 보면 같은 작업을 매번 반복하게 됩니다. 플러그인은 **설치 한 번으로 묶음 재사용**, 그리고 **마켓플레이스를 통한 팀·커뮤니티 공유**를 가능하게 합니다.

**비유:** 앱을 하나씩 따로 까는 대신 **'앱 묶음 패키지'를 한 번에 설치**하는 것. [[20260613_클로드 코드 Skill 반복 자동화]]·[[20260613_클로드 코드 Hook 자동 가드레일]]·MCP를 낱개로 쓰다가, 준비되면 플러그인으로 묶습니다.

**패턴:** `.claude/`에서 먼저 실험 → 준비되면 플러그인으로 변환.

---

## ⚖️ 독립 설정(.claude/) vs 플러그인

| 항목 | `.claude/` | 플러그인 |
|---|---|---|
| 스킬 이름 | `/hello` | `/plugin-name:hello` |
| 적합한 용도 | 개인 워크플로 · 빠른 실험 | 팀 공유 · 버전 관리 |
| 공유 방식 | 수동 복사 | 마켓플레이스 설치 |
| 네임스페이스 | 없음 · 충돌 가능 | 자동 네임스페이싱 |

즉 **혼자 빠르게 실험할 땐 `.claude/`, 남들과 나누고 버전 관리할 땐 플러그인**입니다.

## 🧱 플러그인 구조 — 위치가 곧 인식 조건

정해진 디렉토리 규칙이 있고, **파일을 올바른 위치에 둬야 인식**됩니다.

```
my-plugin/
├── .claude-plugin/      # plugin.json '만' 배치
│   └── plugin.json
├── skills/              # <name>/SKILL.md
├── commands/            # 슬래시 커맨드 .md
├── agents/              # 서브에이전트
├── hooks/               # hooks.json
├── monitors/            # 백그라운드 모니터
├── themes/              # 컬러 테마
├── scripts/             # 훅·유틸 스크립트
├── settings.json
├── .mcp.json            # MCP 서버
└── .lsp.json            # LSP 서버
```

**⚠️ 절대 규칙 2가지:**
- `commands · agents · skills · hooks · monitors · themes`는 **플러그인 루트**에 배치
- `.claude-plugin/` 안에는 **`plugin.json`만** 둔다

> ⚠️ **검토 요청:** 위 디렉토리 규칙·버전 노트(예: 루트에 `SKILL.md` 하나만 있으면 플러그인 자체가 단일 스킬로 노출되는 '단일 스킬 모드' 등)는 도구 버전에 따라 바뀝니다. 실제 제작 전 최신 문서·버전을 확인하세요.

## 🔌 실전에서 자주 쓰는 플러그인 예시

| 플러그인 | 성격 | 포함/제공 |
|---|---|---|
| **Superpowers** | 종합 생산성 번들 | 코드 리뷰·테스트 자동화 스킬, 파일 보호·lint 훅, 리서치·디버깅 서브에이전트 |
| **gstack** | Git 워크플로 | `/gstack:new`(새 스택), `/gstack:submit`(다단 PR), `/gstack:sync`(메인 반영) |

설치는 보통 `/plugin install <이름>` 형태로 합니다.

> ⚠️ **검토 요청:** 플러그인 이름·명령·설치 방식은 생태계 변화가 빨라 자주 바뀝니다. 실제 설치 전 최신 마켓플레이스·문서를 확인하세요.

## 🧪 직접 해보기 (개념 체크)

1. 내가 `.claude/`에 만들어 둔 스킬·훅이 있다면, **"이걸 팀과 공유하려면?"**을 떠올려 본다 → 플러그인 후보
2. 혼자 실험 단계인지(`.claude/`), 공유·버전 관리 단계인지(플러그인) 구분해 본다
3. Superpowers 같은 기성 플러그인이 내 반복 작업을 이미 해결해 주는지 살펴본다

## 💡 핵심 교훈

- 플러그인은 **스킬·훅·MCP·에이전트를 한 패키지로 묶어 재사용·공유**하는 단위입니다.
- **혼자 실험은 `.claude/`, 팀 공유·버전 관리는 플러그인** — 플러그인은 자동 네임스페이싱으로 충돌을 막습니다.
- 핵심은 `.claude-plugin/plugin.json` 매니페스트, 그리고 **올바른 폴더 위치 = 인식 조건**입니다.
