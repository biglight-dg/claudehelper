# 클로드 코드 Slash 명령어와 Plan Mode — 안전하고 똑똑하게 시키기

## 🎛️ 클로드 코드를 '제대로' 부리는 두 장치

클로드 코드에 한국어로 일을 시키는 건 배웠습니다. 이번엔 **더 똑똑하고 안전하게** 쓰는 두 장치 — **Slash 명령어**(빠른 단축 명령)와 **Plan Mode**(읽기 전용 설계 모드) — 그리고 **권한(Permission) 4단계**입니다.

---

## ⌨️ Slash 명령어 — 자주 쓰는 10개

`/`로 시작하는 단축 명령입니다. 외울 필요 없이, 자주 쓰는 것만 알아두세요.

| 명령 | 기능 | 명령 | 기능 |
|---|---|---|---|
| `/login` | 계정 인증 | `/init` | CLAUDE.md 자동 생성 |
| `/context` | 토큰 사용 현황 | `/compact` | 대화 압축 |
| `/clear` | 대화 초기화 | `/cost` | 비용 확인 |
| `/permissions` | 권한 관리 | `/model` | 모델 변경 |
| `/statusline` | 상태 표시줄 설정 | `/help` | 전체 명령어 목록 |

**핵심:** `/init`(지침 자동 생성), `/context`(토큰 점검), `/clear`(새 주제 시작) 세 개만 손에 익혀도 충분합니다.

---

## 🚦 권한(Permission) 4단계 — AI에게 어디까지 허용할까

클로드가 내 파일을 직접 다루므로, **얼마나 자동으로 할지**를 4단계로 고릅니다.

| 모드 | 성격 | 언제 |
|---|---|---|
| **Ask Before Edits** | 모든 수정 전 확인 (가장 안전) | 첫 사용·학습 단계 |
| **Edit Automatically** | 기존 파일 자동 수정, 새 파일만 확인 | 일상 작업(권장) |
| **Plan Mode** | 읽기만 가능 (수정 불가) | 복잡한 설계 전 탐색 |
| **Bypass Permissions** | 모든 작업 자동 (위험) | 본인 PC 자동화 한정 |

**비유:** 운전 면허 단계 같아요. 처음엔 강사가 옆에서 매번 확인(Ask), 익숙해지면 일상 주행(Edit), 위험 구간은 천천히 살피기(Plan), 무인 자동은 통제된 곳에서만(Bypass).

> ⚠️ **Bypass 위험:** 잘못된 명령 해석으로 파일 삭제·데이터 손실 위험. Sandbox·인터넷 차단 환경에서만 권장합니다.

---

## 📐 Plan Mode — 짓기 전에 설계도부터

**Plan Mode**는 **읽기만 하고 실제로 바꾸지 않는** 탐색 모드입니다. 복잡한 작업은 여기서 계획부터 세웁니다.

| ✅ 가능 | ❌ 불가능 |
|---|---|
| 파일 읽기 | 파일 수정 |
| 웹 리서치·구조 분석 | 새 파일 생성 |
| 구현 계획(plan) 작성 | 실제 빌드 실행 |

**비유:** 건축 설계도 단계. **설계도에서 고치면 저렴**하지만, 다 짓고 부수면 비쌉니다. 코드도 똑같아요.

---

## 🔄 Plan Mode 실전 흐름 + 효과

**7단계 워크플로우:** ① 요구사항 음성/메모로 쏟아내기 → ② Plan Mode 진입 → ③ **클로드의 되묻기(Clarifying Questions)에 답** → ④ 구현 계획 검토 → ⑤ Edit/Bypass 모드로 전환 → ⑥ 실제 빌드 실행.

3번 **'되묻기'가 핵심** — 클로드가 모호한 부분을 먼저 질문해 **잘못된 가정을 사전 차단**합니다.

| 구분 | 소요 시간 | 결과 |
|---|---|---|
| Plan 없이 바로 | 35분+ | 토큰 낭비 큼, 재작업 |
| Plan 먼저 | 15~25분 | 안정성 ↑, 절약 |

## 💡 핵심 교훈

- **Slash 명령어**는 단축키 — `/init`·`/context`·`/clear` 정도만 익히면 충분.
- **권한 4단계**로 AI 자율성을 조절: 학습엔 Ask, 일상엔 Edit, 설계엔 Plan, 자동화는 Bypass(위험).
- 복잡한 일은 **Plan Mode로 설계도부터** — '되묻기'로 가정을 바로잡으면 시간·토큰이 크게 절약됩니다.
